# PPT_GUIDE

## Deck Brief

- Title: `Meet Claude Code — your patient AI coworker`
- Goal: `Make a total beginner (non-programmer, possibly non-CS) feel that Claude Code is approachable, useful on day one, and vaguely magical — in 6 pages.`
- Audience: `Humans who have never written a line of code. No CS background required.`
- Duration: `60 seconds of scroll-through, or a 2-minute conversation.`
- Slide Budget: `6`
- Presenter: `The README`
- Aspect Ratio: `16:9`
- Tooling Backend: `Python / python-pptx`
- Notes Policy: `script-only` (notes copy the read-aloud line; no extra supplements)
- Audience Language: `English`
- Voice: `Playful, confident, self-aware. Slightly irreverent but never cynical. Never condescending.`
- Style Constraints:
  - `Modern. Lots of whitespace. No clipart. No shadows.`
  - `Maximum two fonts. Monospace only for code/terminal.`
  - `Warm, saturated accent color. Avoid corporate blue.`
  - `Every page is readable at thumbnail size for the README preview.`
- Source Materials:
  - `None — this is a vibes-first intro, not a feature list.`
- Acceptance Criteria:
  - `Rendered PNG grid looks good in README at ~900px wide.`
  - `Each slide has exactly one dominant idea.`
  - `Each slide has at least one line with a smile in it.`

## Document Purpose And Usage

- This guide is the source of truth for deck intent.
- The deck lives at `assets/claude-code-intro/` inside the deck-workflow-skill repo.
- The deck is a self-demo: it is generated by applying the skill to itself.

## Global Production Principles

- One main idea per slide.
- Titles carry opinion, not just section names.
- No corporate jargon. No "empowering your workflow".
- Prefer concrete examples to abstract features.
- Keep code snippets in DejaVu Sans Mono so indentation is honest.
- Cover and closing have the boldest color treatment; inner slides lean clean-and-roomy.
